# Dice Batch Grid

4x4盤面に赤・青・黄・緑のサイコロ（数字1〜4）をドラッグ配置するブラウザパズルです。既存ゲームの名称・ロゴ・画像・音声・文章は使用していません。

## 基本ルール

- ROLLで現在のバッチとして4個のサイコロを生成します。
- バッチ4個がすべて盤面で固定されるまで、新しい4個は補充されません。
- バッチの一部だけを盤面に置いてROLLすると、その未固定サイコロだけを固定し、役判定と消去処理を行います。
- 未固定サイコロはPC/スマホのドラッグで自由に移動できます。固定済みサイコロは鍵表示になり移動できません。
- 役判定対象は横4列、縦4列、斜め2本、2x2ブロック9個、四隅1個の合計20グループです。

## ジョーカー

- ROLLでは出現しません。
- 役得点が800点貯まるごとに1個獲得します。
- 最大ストックは2個で、ジョーカー欄からドラッグして盤面に置けます。
- ジョーカー表示には `assets/dice/JK.png` を使用します。
- 役判定では任意の色・数字として扱うワイルドカードです。

## 画像ファイル

PNG画像はこのPRには含めません。運用時はGitHubのmainブランチに手動で以下のファイルを `assets/dice/` 配下へ配置してください。

- 青: `b1.png`, `b2.png`, `b3.png`, `b4.png`
- 赤: `r1.png`, `r2.png`, `r3.png`, `r4.png`
- 黄: `y1.png`, `y2.png`, `y3.png`, `y4.png`
- 緑: `g1.png`, `g2.png`, `g3.png`, `g4.png`
- ジョーカー: `JK.png`

コード側はこれらのファイルが `assets/dice/` に存在する前提で参照します。

## ゲームオーバーとランキング

ROLLで固定した後、消去が発生せず盤面に空きマスがなくなるとゲームオーバーです。名前とスコアは端末のlocalStorageに上位10件まで保存されます。ハイスコアもlocalStorageに保存されます。

共有ランキングを実装する場合は、ブラウザにGitHubトークンを埋め込まず、Google Apps Script + Googleスプレッドシートを外部APIとして使う方式が安全で簡単です。現在のランキング処理はlocalStorage用と将来の外部API用に関数分離しています。

## 点数

- 同色同数字4個: 240（消去）
- 同色1-4: 200（消去）
- 4色同数字: 180（消去）
- 4色1-4: 160（消去）
- 色4個 / 数字4個: 40
- 色2色x2個 / 数字2種類x2個: 25
- 4色 / 数字1-4: 20

## 開発

```bash
npm test
npm start
```
